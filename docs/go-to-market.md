# RenJistroly 产品发布与市场策略

## v0.2.0 发布策略

### 定位

**"Codex 是 coding agent，RenJistroly 是 Mac operating agent"**

RenJistroly 的差异点不在代码补全，而在操控整个 Mac 工作环境。基于企业内部 200+ 真实场景推导的功能矩阵覆盖了工程、设计、运营、安全四个域，而 Codex 等竞品仅覆盖工程域。

### 目标用户分层

| 阶段 | 用户 | 触达方式 |
|------|------|----------|
| v0.2 首发 | macOS 开发者 / 工程师 | GitHub Releases、开发者博客 |
| v0.3 扩展 | 设计师 / PM / 运营 | 产品 Hunt、短视频演示 |
| v1.0 规模 | 非技术高管 | 企业销售、MDM 渠道 |

### 核心卖点

- **10 种安全模式 vs 竞品 0**：从只读到完全控制，覆盖企业内部合规审计要求。
- **企业就绪**：内置角色管理、操作审计、策略引擎，开箱即用。
- **中文优先**：中文 UI、中文文档、中文场景深度覆盖。

### 发布渠道

1. **GitHub Releases** — 开源社区引流，配合 Release Notes 展示每个版本的场景矩阵扩展。
2. **Product Hunt** — 面向设计师/PM 群体的产品发布，30 秒操作演示短视频。
3. **macOS 开发者博客** — 技术向深度文章，对比安全模型与架构设计。
4. **Twitter/X 短视频** — 30 秒功能演示，突出"agent 操作整个 Mac"的视觉冲击。

### 路线图

- **v0.2.0**（当前）：企业安全 + 角色场景 + 测试矩阵覆盖 80%+
- **v0.3.0**：语音交互 + 实时协作
- **v0.4.0**：插件系统 + 第三方集成
- **v1.0**：稳定企业版 + MDM 大规模部署

### 度量指标

| 指标 | 目标 | 周期 |
|------|------|------|
| GitHub Stars | 千星 | 6 个月 |
| 测试覆盖率 | 80%+ | 持续 |
| 企业 POC | 5+ | 3 个月 |
| 场景矩阵覆盖 | 200+ | v1.0 |

### 执行优先级

短期：完善 GitHub 仓库文档与 Release 流程，确保开发者能 5 分钟完成安装试用。中期：产出 3 篇深度技术博客 + 2 条操作演示视频，分别面向开发者和产品决策者。长期：建立企业销售 Pipeline，通过 MDM 集成进入企业 IT 采购目录。
