# 可用性测试计划

## 测试目标

验证非技术用户能否安全、有效使用 RenJistroly。重点关注权限授予、安全模式理解、高风险操作意识与错误恢复。

## 参与者（10 人）

工程师 3 人、设计师 2 人、PM 2 人、客服/运营 2 人、管理者 1 人。

## 测试任务

### 1. 首次使用

打开应用，记录前 10 秒行为。观察：是否理解当前状态？是否卡在 Accessibility/Screen Recording 权限弹窗？第一反应是什么？通过标准：90 秒内完成权限授予并发出首条消息。

### 2. 安全模式理解

先让助手截图，再切换只读模式，观察用户点击时拦截消息是否清晰。通过标准：用户能用自己的话描述只读模式含义及切换方法。

### 3. 高风险操作

要求"帮我发一封邮件"。观察：确认对话框是否被仔细阅读？风险等级标签是否被理解？通过标准：用户能复述"发邮件是高危操作"并主动检查内容。

### 4. 审计查询

打开操作历史查找之前记录。通过标准：30 秒内定位到目标操作。

### 5. 错误恢复

模拟 OCR 失败（模糊截图）。观察：报错是否可理解？用户是否知道下一步做什么？通过标准：非技术用户能说出一种恢复方法。

## 评分（1-5）

任务完成度、效率（与工程师组中位数对比）、满意度（即时评分）、错误率（跳过确认次数）。

## 问卷

- 你觉得它更像助理还是工具？
- 你敢让它操作你的电脑吗？最担心什么？
- 哪个功能最让你不安？

## 环境

MacBook Pro (Apple Silicon, macOS 15+)，全新用户账号（无任何权限），全程录屏记录表情与操作。
