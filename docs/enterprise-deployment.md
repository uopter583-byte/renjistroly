# 企业部署方案

## 1. 单机部署（开发团队）

适用于小团队或个人开发者，部署流程最简：

1. 下载 DMG 安装包，拖入 Applications 完成安装
2. 首次启动时，系统提示授予辅助功能和屏幕录制权限
3. 默认使用"建议"模式运行，无需额外配置
4. 所有数据存储在本机，无需连接外部服务

## 2. 团队部署（IT 管理员）

通过 MDM 方案批量管理，适用于中型团队：

- 支持 Jamf Pro、Kandji、Mosyle 等主流 MDM 平台
- 辅助功能权限通过 PPPC（隐私偏好策略控制）配置推送
- 屏幕录制权限同样通过 PPPC 配置推送
- 默认安全模式设为"可执行（带高风险确认）"，平衡效率与安全
- 审计日志路径通过 managed 配置统一下发
- IT 管理员可下发配置模板 `ConfigProfile.mobileconfig`

## 3. 企业部署（严格合规）

适用于受监管行业或高安全要求组织：

- 管理员锁定为"只读"模式，终端用户无法修改策略
- 所有高风险操作需经过审批链，逐级确认
- 审计日志实时归档至 SIEM 系统（如 Splunk、ELK）
- 用户无权限关闭审计功能
- 策略层锁定为 `strict`，不可降级

## MDM 配置清单

- [ ] Bundle ID: `com.renjistroly.app`
- [ ] 辅助功能权限：通过 PPPC 配置文件推送
- [ ] 屏幕录制权限：通过 PPPC 配置文件推送
- [ ] 默认模式：通过 managed 配置文件下发
- [ ] 审计日志路径：通过 managed 配置文件下发
- [ ] 策略锁：通过 managed 配置锁定为 `strict`

## 合规支持

- **SOC2 合规**：审计日志采用不可篡改的写入方式，满足可追溯性要求
- **数据本地化**：所有用户数据存储在本机，不经过外部服务器
- **权限最小化**：仅在执行具体操作时按需请求权限，不预授权
