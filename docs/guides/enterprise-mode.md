# 企业安全模式

RenJistroly 提供 10 种可叠加的操作模式，覆盖企业合规审计的所有场景。模式可以在设置中的「模式控制面板」开启/关闭。

## 10 种操作模式

| 模式 | 风险等级 | 说明 |
|------|----------|------|
| **只读** | 低 | 所有写操作被拦截，仅允许读取。适合展示、审计场景 |
| **建议** | 低 | 仅提供建议，不执行任何操作。适合决策参考场景 |
| **可执行** | 中 | 允许执行标准操作。日常使用的默认模式 |
| **高风险确认** | 高 | 高风险操作（写文件、Shell、拖拽等）需要用户确认 |
| **禁止鼠标** | 高 | 禁用鼠标控制，防止误操作或脚本驱动点击 |
| **本地模式** | 中 | 禁止所有网络调用，数据完全本地处理 |
| **敏感 App 防护** | 高 | 禁止读取敏感应用（如密码管理、银行）的内容 |
| **自动遮蔽** | 高 | 密码、API Key 等敏感字段自动遮蔽后再输出 |
| **策略锁定** | — | 管理员锁定当前策略配置，不可修改 |
| **审计导出** | 低 | 操作记录导出模式，用于合规审计 |

## 工具风险等级

每个 MCP 工具都有风险等级，决定自动执行还是需要确认：

| 等级 | 行为 | 示例工具 |
|------|------|----------|
| 无 | 自动执行 | 读取屏幕、系统信息 |
| 低 | 可自动 | 列出文件、Git status、打开应用 |
| 中 | 需确认 | 点击元素、按键、菜单导航 |
| 高 | 严格确认 | 写入文件、Shell 命令、拖拽 |

## 执行策略预设

在设置中可以切换三种全局策略：

- **默认** — 低风险自动，中高风险确认
- **宽松** — 低中风险自动，仅高风险确认
- **严格** — 全部需要确认

## 操作确认流程

当 AI 发起高风险操作时：

1. 弹出确认对话框，显示：
   - 工具名称
   - 风险等级
   - 操作摘要
2. 用户选择「批准」或「取消」
3. 批准后 AI 执行操作，执行后验证结果
4. 操作记录进入审计日志

## 审计与追踪

所有操作（包括自动执行和确认执行）都会被记录到审计日志，包括：

- 执行时间
- 工具名称和参数
- 风险等级
- 用户确认状态
- 执行结果
- 上下文快照

审计日志可在 Agent Console 中实时查看，也可导出用于合规审查。

## 典型企业场景

### 开发者桌面
- 启用：可执行 + 高风险确认
- 可自由编码，敏感操作需确认

### 设计师工作站
- 启用：可执行 + 禁止鼠标（防误点）
- 通过键盘快捷键和命令操作

### 公共演示机
- 启用：只读 + 审计导出
- 访客只能查看，所有操作被记录

### 敏感数据处理
- 启用：本地模式 + 自动遮蔽 + 敏感 App 防护
- 数据不出本机，敏感信息自动遮蔽
