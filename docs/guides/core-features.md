# 核心功能

## 桌面操控

RenJistroly 能像人一样操控 Mac 桌面，基于三层回退引擎：

1. **辅助功能 (AX API)** — 直接读取 UI 元素树，点击、输入、滚屏
2. **AppleScript + JS 注入** — 在浏览器中操控 DOM 元素
3. **截图 + OCR 视觉定位** — 当 AX/DOM 都不可用时，通过截图分析找元素

### 支持的操作

| 操作 | 示例指令 |
|------|----------|
| 打开应用 | "打开 Xcode" |
| 点击按钮 | "点击保存按钮" |
| 输入文字 | "在搜索框输入...然后按回车" |
| 滚动页面 | "向下滚动" |
| 拖拽文件 | "把桌面这个文件拖到项目目录" |
| 按键组合 | "按 Command+S 保存" |
| 窗口管理 | "把这个窗口放到左半屏" / "最小化窗口" |

## 屏幕理解

| 能力 | 说明 |
|------|------|
| OCR 文字识别 | 双引擎（Vision + PPOCRv6），支持中/英/日混合 |
| UI 树读取 | 获取当前应用的完整界面结构、元素角色、标签 |
| 截图分析 | 对截图进行视觉理解 |

### 支持的命令

- "读取当前屏幕内容"
- "这个按钮在哪里？"
- "屏幕上有什么弹窗？"

## 浏览器操控

支持 Chrome 和 Safari 双浏览器，通过 DOM 级别的 CSS 选择器操控页面。

| 命令 | 说明 |
|------|------|
| `webpage_structure` | 获取页面 DOM 结构 |
| `dom_click` | CSS 选择器点击元素 |
| `dom_fill` | 填充表单字段 |
| `dom_submit` | 提交表单 |
| `browser_navigate` | 前进/后退/刷新 |
| `get_browser_state` | 获取当前标签页信息 |

### 适用场景

- 自动填写表单
- 批量操作网页
- 网页数据提取
- UI 回归检查

## 多后端 LLM

支持 7 种 LLM 后端，SmartRouter 根据任务复杂度自动路由：

| 后端 | 类型 | 适用场景 |
|------|------|----------|
| Anthropic Claude | 云端 | 复杂推理、多步操作 |
| OpenAI | 云端 | 通用对话 |
| Google Gemini | 云端 | 长上下文 |
| DeepSeek | 云端 | 性价比场景 |
| OpenAI Compatible | 云端 | 兼容服务 (Qwen/Moonshot 等) |
| MLX 本地 | 本地 | 离线、数据敏感场景 |
| Claude Code CLI | 本地 | 开发任务委派 |

## 场景快捷操作

浮动面板底部的快捷按钮：

- **润色** (✨) — 选中文字后一键润色
- **解释** (💬) — 解释选中的代码或文字
- **读屏** (👁) — 读取当前屏幕并分析内容

## 开发者模式

在设置中开启开发者模式后，额外获得：

- 一键构建/测试按钮
- Claude Code 启动器（委派开发任务给 Claude Code）
- OCR 引擎选择
- Self-Update 管理（Helper 状态、签名验证）
